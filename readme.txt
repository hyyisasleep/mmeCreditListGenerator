lib.csv是储存数据的文件，可以用excel打开和修改，目前保存的是我自己登记的mme数据

一些QA：

1.添加表项时文件夹名（dirname）写什么：
	
	只是自己用的话可以用程序默认给出的绝对路径，想共享给别人/以后会挪文件位置的话推荐删掉前缀保留最后一或二级目录
	
2.能不能提前取消生成，一个个点忽略也好麻烦：

	点窗口右上角的叉可以取消，但是这次生成中之前添加到库中的数据也不会保存

3.mme拖进工程里但是取消打勾了没用上怎么算：
	
	从.emm文件里看不出来所以这个生成程序算你用了，可以自己删掉这项或者假装你用了（

4.如何修改作者名/从生成列表中删除没用上的mme：
	
	出结果后点击详细信息按钮后在弹出的表格里可以双击表项修改，双击表中最右列可以删除对应行
	
	（不是从库里删掉只是让这次生成的借物表里不写这个mme了，但是修改的结果是会直接同步到库里的）
	
	我是彩笔所以用了这么离谱的方式做表格（）对不起
	
	每次打开emm文件后程序会读一次lib.csv，在生成结果和保存了修改后的详细结果的时候都会写一次csv文件，
	
	所以修改csv文件后想让它生效需要重新点一次读取emm文件，这期间在程序里改了内容的话小心被写回覆盖（应该碰不到这坑吧，总之写一下
	
5.怎么还有乱码：

	我被编解码搞晕了开摆了，打开emm的时候先用一次中文的GB18030，报错就换shift-JIS再打开一次，最后成功识别出来的路径是什么就保存什么
	
	会导致同一个mme需要登记两次，也在库里存出乱码版和正常日文版两项（比如现在库里就有 嬻拞偵昚偆毢.x 和 空中に漂う埃.x 两个），但能分别匹配成功正常运行就算赢！（
	
	至于乱码版的模型名和场景名，应该能自己认出来吧……

6.匹配规则怎么写的：

	也摆烂了，写的比较弱智, 举个例子：

	在库里登记一个绝对路径为 "D:\XXX(上级目录)XXX\AAA"或者相对路径"AAA"的mme之后,（这个AAA是最后一级文件夹 

	路径是"D:\XXX(上级路径)XXX\AAA"里的所有.x 和.pmx文件都会被自动识别，

	但是"D:\XXX(上级目录)XXX\AAA\BBB" 和"D:\XXX(上级目录)XXX"中的.x和.pmx就要手动重新登记

7.关于多作者判定问题：
	
	有些老师会在rm里注明参考了xxx老师的mme，这个我也不知道怎么界定范围，姑且按文件夹里有几个人的rm就填几个人的名字处理了，不妥的话欢迎指出orz
		
	（如果发现我库里写错作者名/写漏了也非常欢迎指出甚至骂我，我先滑跪
